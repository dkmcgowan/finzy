# Finzy icons – where they come from and where they go

## Single source: `assets/`

All icon **sources** live in **`assets/`**. Nothing else needs to be copied by hand.

| File | Size | Who uses it |
|------|------|-------------|
| **finzy.svg** | — | Master source. Edit this, then regenerate the PNGs. |
| **finzy.png** | 1024×1024 | App UI (auth, about), `flutter_launcher_icons`, Linux packaging. |
| **finzy_android_foreground.png** | 1440×1440 (icon centered) | Android adaptive launcher icon (foreground layer). |
| **finzy_monochrome.png** | 1440×1440, monochrome | Android adaptive launcher (monochrome / themed icon). |

`pubspec.yaml` points at these three PNGs. **flutter_launcher_icons** reads them and generates the actual OS-specific icon files (Android `mipmap-*`, Windows, Linux, etc.) when you run it or build. You do **not** put icons into `android/.../res` or Windows/Linux icon folders by hand for the launcher.

---

## Regenerating the 3 PNGs from the SVG

### Option A: No extra software (recommended if you’re wary of installers)

Use **GitHub Actions** so nothing image-related is installed on your machine:

1. Push your updated **finzy.svg** (and the script) to GitHub.
2. Go to **Actions** → **“Generate icon PNGs”** → **Run workflow** → Run.
3. When it finishes, open the run → **Artifacts** → download **finzy-icon-pngs.zip**.
4. Unzip and copy the three PNGs into your repo’s **assets/** folder (overwrite the existing ones).

You only need Python + Pillow locally if you want to run the script yourself; for the icons, the runner has ImageMagick.

### Option B: Local – use a package manager (no random installers)

To run **`python scripts/generate_finzy_pngs.py`** locally with minimal risk:

- **Windows (winget):**  
  `winget install ImageMagick.ImageMagick`  
  Single package from Microsoft’s store; no third-party installer with toolbars.
- Then: `pip install Pillow` and run the script.

Avoid downloading ImageMagick from random “download” buttons; use winget (or Chocolatey if you already use it).

## What `scripts/generate_android_icons.sh` does (different job)

That script does **not** create the three PNGs above. It:

- Reads **assets/finzy.svg**.
- Produces **Android-only** files and writes them straight into **android/app/src/main/res/**:
  - **drawable-*/ic_stat_notification.png** – white silhouette for the status bar / notifications.
  - **mipmap-*/ic_launcher_monochrome.png** – monochrome launcher icon at several densities.

So:

- **App launcher icon** (and Windows/Linux icons) → from **assets/** via **flutter_launcher_icons** (using the 3 PNGs).
- **Android notification icon** and **Android monochrome launcher** (optional, from SVG) → **generate_android_icons.sh** (needs Bash, rsvg-convert, ImageMagick; typically used on macOS/Linux).

---

## Summary

- **Only place you maintain PNGs:** **`assets/`** (the 3 files above).
- **Regenerate those 3 from SVG:** `python scripts/generate_finzy_pngs.py`.
- **OS-specific launcher icons:** Generated by **flutter_launcher_icons** from those assets; no manual copying.
- **After changing the PNGs** you must **regenerate** platform icons, then rebuild:
  1. `python scripts/generate_finzy_pngs.py` (if you changed the SVG)
  2. **`dart run flutter_launcher_icons`** ← creates/updates `windows/runner/resources/app_icon.ico` and other platform icons
  3. `flutter clean && flutter build -d windows` (or your target)
- **Android notification + monochrome from SVG:** Optional; use **generate_android_icons.sh** if you want them built from the same SVG (e.g. on Mac/Linux).
